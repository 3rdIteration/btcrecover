Start with the conservative default: use --addr-limit 10. For one missing BIP39 word you don’t need placeholders — seedrecover will try all words automatically. Raising --addr-limit to 100 greatly increases work and runtime and is only warranted if you know the target address was generated much later (e.g., you created many addresses after the one you remember). If the first run finds nothing, then try a larger limit like 100 as a second attempt. Do you know roughly how many addresses were generated before that address?
